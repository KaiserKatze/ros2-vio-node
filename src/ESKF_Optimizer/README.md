## 1. 引言

### 1.1 项目标识

- **项目名称**：ESKF超参数自动调优工具（ESKF Hyperparameter Auto-Tuner, EHAT）
- **项目编号**：EHAT-2026-001

### 1.2 背景

在单目视觉惯性里程计（VIO）系统中，误差状态卡尔曼滤波（ESKF）的**观测误差协方差矩阵**置信度超参数对最终轨迹精度具有显著影响。当前用户通过手动调整该超参数，已在仿真数据集上获得多条肉眼近似真实的估计轨迹。然而，手动调试效率低，且超参数取值范围内存在多个“良好”区域，需要一种自动化搜索方法，在给定优化目标下找到最优超参数。

### 1.3 项目目标

开发一个软件系统，该系统能够：

- 接收仿真数据集提供的真实轨迹文件（ground truth）和ESKF算法输出的估计轨迹文件。
- 在用户指定的超参数搜索空间内，利用**贝叶斯优化**算法自动搜索最优的超参数配置。
- 优化目标为“综合误差”，该误差是绝对轨迹误差（ATE）与相对位姿误差（RPE）的加权平均值，其中ATE计算强制使用SE(3)变换（禁止SIM(3)变换）。
- 记录每次评估的超参数组合及其对应的ATE、RPE和综合误差，输出为带表头的CSV文件，便于用户进行可视化数据分析。
- 输出最优超参数及其对应的综合误差值。

### 1.4 适用范围

本软件适用于单目视觉惯性里程计系统的离线超参数调优场景，尤其适用于仿真环境下的ESKF观测误差协方差置信度参数优化。超参数为若干个标量，其个数由用户在配置文件中自由指定（1维至10维）。软件以命令行工具形式运行，不依赖图形用户界面（GUI）。

### 1.5 术语与定义

| 术语 | 定义 |
|------|------|
| **ESKF** | 误差状态卡尔曼滤波（Error-State Kalman Filter），用于VIO状态估计。 |
| **观测误差协方差置信度超参数** | 控制ESKF中观测噪声协方差矩阵大小的乘性因子（标量或向量），记作 `λ`。该参数越大表示观测置信度越低。超参数是若干个标量，其个数由用户在配置文件中定义。 |
| **真实轨迹** | 仿真数据集提供的无误差位姿序列，包含时间戳和6自由度位姿（SE(3)）。 |
| **估计轨迹** | ESKF算法输出的位姿序列。 |
| **ATE** | 绝对轨迹误差（Absolute Trajectory Error），基于对齐后估计位姿与真实位姿之间的均方根误差（RMSE）。 |
| **RPE** | 相对位姿误差（Relative Pose Error），基于固定时间间隔（或距离间隔）的位姿增量误差的RMSE。 |
| **Umeyama算法** | 用于估计两个点集之间最优相似变换（SE(3)或SIM(3)）的最小二乘算法。 |
| **SE(3)变换** | 刚性变换（旋转+平移），不包含尺度缩放。 |
| **SIM(3)变换** | 相似变换（旋转+平移+尺度缩放）。 |
| **综合误差** | `W_ATE * ATE + W_RPE * RPE`，其中 `W_ATE + W_RPE = 1`，默认各0.5。 |

### 1.6 参考文献

- Umeyama, S. (1991). Least-squares estimation of transformation parameters between two point patterns.
- Sturm, J. et al. (2012). TUM RGB-D benchmark for evaluating SLAM systems.
- Snoek, J., Larochelle, H., & Adams, R. P. (2012). Practical Bayesian optimization of machine learning algorithms.
- EVO documentation: https://github.com/MichaelGrupp/evo

## 2. 总体描述

### 2.1 用户角色

| 角色 | 描述 |
|------|------|
| **算法调优工程师** | 负责配置搜索参数、运行调优、分析结果。 |
| **系统集成者** | 将本工具集成到VIO仿真或评估流程中。 |

### 2.2 用户需求与特性

- **需求 U1**：用户应能够通过配置文件指定ESKF超参数的搜索范围。超参数是若干个标量，个数自由定义（例如1个、3个、5个），每个超参数需指定名称、类型（连续或离散）、取值范围。
- **需求 U2**：用户应能够指定ATE与RPE的权重系数（浮点数，非负，和为1）。
- **需求 U3**：系统默认使用**贝叶斯优化**算法；同时应支持扩展其他算法（如随机搜索、网格搜索、遗传算法）。
- **需求 U4**：用户应能够设置优化停止条件（如最大迭代次数、时间限制、目标误差阈值）。
- **需求 U5**：系统应输出最优超参数及其综合误差，同时**必须**输出一个CSV文件，记录每次评估的超参数组合及其对应的ATE、RPE、综合误差（包含清晰表头），以便用户进行可视化分析。
- **需求 U6**：禁止使用SIM(3)变换进行ATE计算，强制执行SE(3)对齐。

### 2.3 假设与依赖

- **假设 A1**：仿真数据集提供的真实轨迹和ESKF估计轨迹文件采用相同的格式（例如TUM格式：`timestamp tx ty tz qx qy qz qw`，或EuRoC格式）。工具应支持常见格式，并可扩展。
- **假设 A2**：ESKF算法允许外部修改观测误差协方差置信度超参数，并可通过命令行或配置文件重新运行。
- **假设 A3**：单次轨迹估计的执行时间相对于优化搜索可以接受（例如小于数秒）。
- **依赖 D1**：系统依赖于现有Python环境（≥3.8）及第三方科学计算库，包括但不限于：`numpy`, `scipy`, `pandas`, `scikit-learn`（或 `GPy`、`GPyOpt`、`Optuna`）用于贝叶斯优化，以及 `evo` 库用于ATE/RPE计算。用户需预先安装这些依赖。

### 2.4 约束条件

- **约束 C1**：ATE计算中，Umeyama算法仅允许SE(3)变换（旋转+平移），必须禁止尺度估计。若ESKF估计轨迹与真实轨迹之间实际存在尺度差异（例如单目VIO未进行尺度校准），用户需自行确保尺度已对齐或接受更大的ATE。
- **约束 C2**：综合误差的加权平均公式中，两者误差单位需一致（均为平移部分的RMSE，单位为米）。RPE可使用平移部分的RMSE或包含旋转的复合误差，本规约默认采用平移部分RMSE。
- **约束 C3**：若用户未指定权重，默认 `W_ATE = 0.5, W_RPE = 0.5`。
- **约束 C4**：超参数维度由用户在配置文件中定义，通常建议为1-10维以保证优化效率；系统应能处理更高维度，但性能可能下降。

## 3. 系统需求

### 3.1 功能需求

#### 3.1.1 输入处理

| ID | 需求描述 |
|----|----------|
| FR-1 | 系统应能够读取配置文件（JSON/YAML/TOML）或命令行参数，包含以下内容：<br> - 真实轨迹文件路径 <br> - 估计轨迹生成方式（可执行文件路径+参数模板，或直接提供已存在的估计轨迹目录）<br> - 超参数列表：每个超参数的名称、类型（float/int/categorical）、取值范围（或离散值列表）。超参数个数由该列表长度决定。<br> - ATE与RPE权重（浮点数，自动归一化）<br> - 优化算法选择（默认贝叶斯优化）与算法特定参数（如贝叶斯优化的采集函数、初始点数目等）<br> - 停止条件（最大迭代次数、时间、目标阈值） |
| FR-2 | 系统应验证输入合法性：文件存在、权重和为1、取值范围有效等；若不合法，给出明确错误信息并退出。 |

#### 3.1.2 超参数搜索与优化

| ID | 需求描述 |
|----|----------|
| FR-3 | 系统必须实现**贝叶斯优化**作为默认优化引擎。实现细节：<br> - 概率代理模型：默认使用**高斯过程**（Gaussian Process），核函数可选Matérn 5/2或RBF。<br> - 采集函数：默认使用**期望改进（EI）**，可配置为UCB或PI。<br> - 初始设计：使用拉丁超立方采样或随机采样生成 `n_init` 个初始点（用户可配置，默认10个）。<br> - 每次迭代根据采集函数选择下一个超参数组合。<br> - 必须支持连续型和离散型混合超参数空间。 |
| FR-4 | 系统应提供优化算法扩展接口，便于用户切换至随机搜索、网格搜索或遗传算法（通过修改配置文件中的 `algorithm` 字段）。 |
| FR-5 | 对于每一组超参数候选值，系统应：<br> - 调用ESKF估计器生成估计轨迹文件（或读取已有轨迹）。<br> - 计算估计轨迹与真实轨迹之间的综合误差（见FR-6）。<br> - 记录超参数组合、ATE、RPE、综合误差（必须保存至CSV文件）。 |
| FR-6 | 综合误差计算步骤必须严格遵循以下流程：<br> 1. 读取真实轨迹文件（位姿序列 `P_gt`）。<br> 2. 读取估计轨迹文件（位姿序列 `P_est`）。<br> 3. 利用Umeyama算法计算从 `P_est` 到 `P_gt` 的最优SE(3)变换（禁止尺度），得到变换后的估计轨迹 `P_est_aligned`。<br> 4. 计算ATE：调用evo的 `evo_ape` 功能，使用 `-a` 参数（仅对齐，不校正尺度），提取平移部分的RMSE（单位：米）。<br> 5. 计算RPE：调用evo的 `evo_rpe` 功能，使用固定时间间隔（用户可配置，默认1秒或1米），提取平移部分的RMSE。<br> 6. 计算综合误差 = `W_ATE * ATE + W_RPE * RPE`。<br> 7. 若计算过程异常（如轨迹长度不一致、对齐失败），则该组适应度设为无穷大（或一个大惩罚值）。 |
| FR-7 | 系统应支持并行评估候选超参数（多进程/多线程），以加速优化。 |

#### 3.1.3 输出与日志

| ID | 需求描述 |
|----|----------|
| FR-8 | 优化结束后，系统应输出：<br> - 最优超参数组合及其综合误差。<br> - 最优超参数对应的ATE和RPE分量值。<br> - **必须**生成一个CSV文件，包含表头，记录所有已评估的超参数组合及其对应的ATE、RPE、综合误差。表头格式示例：`param1, param2, ..., paramN, ATE, RPE, combined_error`。该文件便于用户导入绘图工具（如matplotlib、pandas）进行可视化分析。<br> - 可选：收敛曲线（每一轮的最佳/平均目标值）保存为CSV或绘图文件。 |

| FR-10 | 用户可选择保存所有评估过的超参数及其结果至另一个CSV文件（FR-8已强制要求，此处可选额外的详细日志）。 |

## 4. 优化算法详细设计（贝叶斯优化）

### 4.1 算法选择理由

贝叶斯优化适用于目标函数评估成本高、非凸、导数未知的黑盒优化问题。本场景中每次评估需运行一次完整的VIO仿真（通常耗时秒级），且综合误差曲面可能具有多个局部极小值。贝叶斯优化通过概率代理模型（高斯过程）和采集函数的权衡，能以较少的评估次数找到全局较优解，比网格搜索或随机搜索更高效。

### 4.2 实现框架

建议使用成熟Python库，如 `scikit-optimize` (skopt) 或 `GPyOpt` 或 `Optuna`（支持贝叶斯优化的TPE算法）。本需求规约不强制指定具体库，但要求：

- 支持混合变量（连续 + 离散 + 类别）。
- 支持并行评估（异步或同步）。
- 支持自定义采集函数。

### 4.3 流程

1. **初始设计**：在超参数空间内，使用拉丁超立方采样或Sobol序列生成 `n_initial_points` 个候选点。对每个点调用 `fitness` 获得综合误差。
2. **代理模型拟合**：基于所有已评估点（超参数→综合误差）训练高斯过程（或TPE）。核函数默认Matérn 5/2，噪声项 `alpha=1e-6`。
3. **采集函数最大化**：计算下一个最“有希望”的超参数点（如最大化期望改进EI）。
4. **评估**：调用ESKF生成估计轨迹并计算综合误差，将结果加入数据集。
5. **判断终止**：若达到最大迭代次数或最优值连续 `n_no_improvement`（默认5次）未改进，或综合误差低于阈值，则终止。
6. **返回最优超参数**。

### 4.4 适应度函数

同FR-6，注意禁止尺度变换。

### 4.5 约束处理

- 超参数边界通过优化算法的“搜索空间”定义直接约束。
- 若离散型超参数，则在采集函数建议后进行最近邻映射或使用专用离散变量处理。

### 4.6 终止条件

满足以下任一条件即终止：
- 达到最大迭代次数（`max_iterations`，含初始点）。
- 最优综合误差连续 `patience` 次（默认5次）迭代未下降超过 `tol=1e-5`。
- 最优综合误差低于 `stop_on_error_threshold`。

## 5. 验证与确认

### 5.1 测试用例

| 测试ID | 描述 | 预期结果 |
|--------|------|----------|
| TC-1 | 提供已知最优超参数附近的搜索空间（2维），运行贝叶斯优化 | 系统应在少量评估（<30次）内找到接近该参数的解，且综合误差小于手动最优值。 |
| TC-2 | 设置 `ate_weight=1.0, rpe_weight=0.0` | 系统仅优化ATE，最优超参数应使ATE最小。 |
| TC-3 | 强制使用SIM(3)对齐（不应允许） | 若代码试图使用SIM(3)，应视为错误；测试中应验证未调用该模式。 |
| TC-4 | 轨迹文件时间戳不对齐 | 系统应自动插值或最近邻匹配，不崩溃。 |
| TC-5 | 并行评估 | 与串行结果一致，加速比接近核数。 |
| TC-6 | CSV输出完整性 | 每次评估的记录均写入CSV，表头正确，无缺失值。 |

### 5.2 验收标准

- 能对至少1-5维超参数空间进行有效搜索，贝叶斯优化相比随机搜索能以更少评估次数达到相似或更优误差。
- 在典型的仿真数据集上（例如EuRoC MAV），优化后的综合误差相比手动调参降低至少10%。
- 提供完整的文档和示例。
- 代码开源且通过单元测试。

## 6. 附录

### 6.1 禁止SIM(3)变换的技术保证

在实现ATE计算时，必须调用evo工具或实现Umeyama算法时设置 `allow_scale=False`。例如使用 `evo_ape` 命令：

```bash
evo_ape tum gt.tum est.tum -a --pose_relation trans_part --save_results results.zip
```

其中 `-a` 表示SE(3)对齐（不包含尺度）。若使用 `-as` 则会开启尺度校正，应禁止。

如果自行实现Umeyama，应忽略尺度因子，即强制 `s=1`，仅求解 `R` 和 `t`。

### 6.2 RPE计算建议

推荐使用 `evo_rpe` 命令：

```bash
evo_rpe tum gt.tum est.tum --delta 1 --delta_unit m --pose_relation trans_part
```

### 6.3 超参数搜索空间扩展性

超参数个数由用户在配置文件中定义，系统应动态构建搜索空间对象。对于贝叶斯优化，需要支持变维度的参数空间，推荐使用 `skopt.space.Real`, `Integer`, `Categorical` 构建列表。

### 6.4 依赖库列表（建议版本）

- Python ≥ 3.8
- numpy ≥ 1.21
- scipy ≥ 1.7
- pandas ≥ 1.3
- scikit-optimize (skopt) ≥ 0.9.0 或 GPyOpt ≥ 1.2.6 或 optuna ≥ 3.0
- evo ≥ 1.12.0
- matplotlib（用于绘图）
